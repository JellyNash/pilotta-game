Pilotta Game - Responsive Overflow Test

Test Configuration

This test will check for overflow issues at different resolutions.

The game containers have been updated to use overflow: visible to detect clipping.

Manual Test Instructions

  1. Open the game in a new browser window
  2. Open Developer Tools (F12)
  3. Enable Device Mode (Ctrl+Shift+M or Cmd+Shift+M)
  4. Test these resolutions:
    • Mobile: 375x667 (iPhone SE)
    • Tablet: 768x1024 (iPad)
    • Desktop: 1366x768 (Common laptop)
    • Full HD: 1920x1080
  5. Look for:
    • Horizontal or vertical scrollbars
    • Content being cut off
    • Cards not fully visible
    • UI elements overlapping

Changes Made

Fixed Issues:

  • HTML: Added responsive width and overflow handling
  • Body: Changed from overflow: hidden to overflow: auto
  • #root: Removed fixed dimensions (was 1477x1270px), now uses 100% width and min-height: 100vh
  • .app: Changed from overflow: hidden to overflow: visible
  • .game-content: Changed from fixed height to min-height

Expected Behavior:

  • At small resolutions, scrollbars should appear if content overflows
  • Cards should scale down appropriately
  • No content should be invisibly clipped
  • Layout should adapt to viewport size
